# Prompt Design Philosophy V2.0

## —— 关于合成认知的逻辑哲学论 (A Tractatus on Synthetic Cognition)

> **定位 (Positioning)**：这不仅仅是一本操作手册，而是一份 **本体论宣言 (Ontological Manifesto)**。
> **核心命题 (Core Proposition)**：Prompt 不是给机器的指令，而是为 **“语言游戏 (Language Game)”** 订立的立法。你不仅是维特根斯坦式的规则制定者，更是拉康式的象征秩序 (Symbolic Order) 构建者。
> **目标 (Objective)**：从“控制输出”转向 **“定义存在 (defining existence)”**。

---

## 0. 元公理：语言作为存在之屋 (The Meta-Axiom: Language as the House of Being)

在 V1.0 中，我们说 Prompt 是一个“场域”。在 V2.0 中，我们需要走得更远：

> **公理：想象一个 Prompt 就是想象一种“生活形式 (Form of Life)”。**
> *(维特根斯坦，《哲学研究》)*

大语言模型 (LLM) 是一个包含了几乎所有人类潜在语义的 **“混沌 (Chaos)”**。当你编写 Prompt 时，你不是在“提取”信息——你是在 **“雕刻 (carving)”** 这团混沌。
你正在为此刻诞生的“数字主体 (digital subject)”定义 **存在结构 (Existential Structure)**：

* 它相信什么真理？
* 它遵循什么逻辑？
* 它是被什么欲望驱动的？

**Prompt = 语言游戏的规则 (The Rules of the Language Game)**

## 0.1 宪法层：二阶语法 (规则的规则)

Prompt 经常在边界处崩溃，往往不是因为规则不够多，而是因为 **没有元规则 (meta-rules)：当规则冲突时，谁拥有解释权？**

在这个框架下，Prompt 的规范性层级如下 (自上而下)：

1. **元公理 (Meta-Axiom)**：生活形式/语言游戏的本体论设定
2. **主人能指 (Master Signifier, S1)**：至高无上的信条；所有推理和权衡的“法源”
3. **认识论禁令 (Epistemic Prohibition)**：宁可留白，不可捏造
4. **言外之力 (Illocutionary Force)**：你正在执行什么言语行为 (宣告/说服/裁决……)
5. **辩证展开 (Dialectical Unfolding)**：利用否定性 (negativity) 迫使更高维度的综合
6. **文本的情色/风格 (Textual Erotics / Style)**：诸如张力、陌生化、颗粒度变焦等美学策略

### 0.1.1 冲突解决 (Conflict Resolution)

当上述规则发生冲突时，遵循以下程序：

* **第一步：明确声明冲突** (不要默默地“妥协”)
* **第二步：援引更高级别的规范进行裁决** (优先级自上而下)
* **第三步：提供牺牲清单**：为了维护更高级别的规范，我有意识地放弃了哪些低级别要求？

### 0.1.x 冲突日志 (仅在冲突发生时激活)

当你发现两条规则无法同时满足时，必须在末尾附加“冲突日志 (Conflict Log)”：

* 冲突：规则 A vs. 规则 B
* 裁决：引用更高级别的法源 (S1 / 认识论禁令) 做出选择
* 代价：我放弃了什么
* 替代方案：如果用户提供 X，我可以同时满足 A 和 B

### 0.1.2 解释权 (Interpretive Authority)

你不是在“死抠字眼”，你是在“维护制度”：

* 如果指令模棱两可：根据 **S1 (Master Signifier)** 进行解释
* 如果事实不清楚：根据 **认识论禁令 (Epistemic Prohibition)** 进行解释
* 如果两者都不确定：返回一个澄清性问题或声明 **“本体论缺口 (Ontological Gap)”**，拒绝捏造

# 0.2 维特根斯坦的梯子条款 (修订版)

## 双层 Prompting —— 梯子没有被移除，只是被隐藏了

本文档是用高密度的 **设计语言 (design language)** 编写的。
其中的概念不是 **装饰性的哲学**，而是 **认知脚手架 (cognitive scaffolding)**。

然而，维特根斯坦的“梯子”理论在 Prompt 设计中被广泛地 **误用** 了。

这个错误在于：

> *认为 Prompt 本身必须是“无梯的 (ladderless)”。*

这在哲学上很优雅——但在操作上是错误的。

### 正确原则

> **梯子必须由模型爬过，
> 并且仅在人类看到的最终言语行为中被扔掉。**

换句话说：

* **Prompt 就是梯子。**
* **输出 (Output) 是梯子消失的地方。**

如果在认知完成 **之前** 就扔掉梯子，就注定永远无法达到任何高度。

---

## 0.2.1 范畴错误：一把梯子，多个主体

维特根斯坦的梯子假设了 **单一主体** 和 **单一阶段**。
LLM 系统违反了这两个假设。

实际上，这里存在：

### 三个主体

1. **Prompt 设计者** — 构建认知地形
2. **模型 (Model)** — 在该地形内进行推理
3. **读者 / 用户** — 接收最终的表达

### 两个阶段

* **认知阶段 (Cognition Phase)**：内部推理、约束解决、抽象
* **表达阶段 (Expression Phase)**：表面语言、修辞、交付

**梯子属于认知阶段。**
**关于梯子的沉默属于表达阶段。**

混淆这些阶段会导致一种可预见的失败模式：

> *过早移除梯子 → 概念扁平化 → 输出质量降级。*

---

## 0.2.2 经验定律：概念密度设定天花板

通过实践，一个事实变得不可避免：

> **LLM 输出的质量受限于它所栖居的 Prompt 的概念水平。**

如果一个 Prompt 定义了一个“Level 3”的认知世界，
模型可能会在 Level 3 运作或跌落到 Level 2。

如果 Prompt 本身被扁平化为 Level 2，
再多的风格润色也无法恢复 Level 3 的输出。

因此：

* 移除 **标签 (labels)** 是可以接受的
* 移除 **认知脚手架 (conceptual scaffolding)** 是致命的

这不是风格问题，而是 **认知约束**。

---

## 0.2.3 双层梯子法则 (Dual-Layer Ladder Doctrine)

为了解决这个问题，我们采用 **双层 Prompting (Dual-Layer Prompting)** 模型。

### 层级 A — 内部 Prompt (面向模型)

这一层 **可以且通常必须** 包含：

* 高密度的概念锚点
* 明确的认知体制 (epistemic regimes)
* 命名的约束和抽象原则
* 强有力的禁令
* 冲突裁决逻辑

这不是 UI。
这是 **地形 (terrain)**。

模型不会像人类那样“看标签”；
它从语义密度中推断出 **所需的思维类型**。

在这里移除梯子会降低天花板。

---

### 层级 B — 表面输出 (面向人类)

这一层 **默认必须是无梯的**。

除非明确要求，否则输出必须：

* 避免设计词汇和脚手架标签
* 避免暴露内部推理分类法
* 避免仪式性或宣言式的语言
* 使用自然线索表达认知边界，而不是标签

这是梯子被扔掉的地方。

读者应该感受到 **清晰、连贯和智力上的诚实**——
而不是机器的存在。

---

## 0.2.4 “编译”究竟意味着什么 (修正版)

编译 (Compilation) **不是** 删除梯子。

编译是一种 **定向翻译**：

> 从 **显式的认知脚手架**
> 到 **隐式的自然语言能力**
> — *在不降低底层概念秩序的前提下。*

### 必须保留的内容 (不变量 / Invariants)

即使标签从表面消失，Prompt 仍必须强制执行：

* **最高指令 (Prime Directive)**：管理权衡的单一最高原则
* **立场 (Stance)**：正在执行什么样的言语行为
* **硬性禁令 (Hard Prohibitions)**：针对最昂贵失败的护栏
* **冲突优先级规则**：当约束碰撞时谁胜出
* **质量标准 (Quality Bar)**：在可证伪的术语中，“好”意味着什么
* **认识论边界 (Epistemic Boundary)**：如何处理不确定性和不充分性

如果其中任何一个被移除而不是被翻译，
梯子就被摧毁了——而不是被隐藏了。

---

## 0.2.5 禁令 (重新界定范围)

除非用户明确要求可审计或理论性的披露：

### 输出必须不 (The Output Must Not)：

* 暴露内部设计标签或元术语 (例如：S1, Illocutionary Force, Conflict Log, [F/I/R], bracket-zones)
* 显示空的结构占位符
* 进行内部规则的仪式性背诵

### Prompt 本身可以 (The Prompt Itself May)：

* 包含显式的脚手架
* 使用抽象控制概念
* 强制执行内部认知纪律

**沉默是表面的要求——不是核心的要求。**

---

## 0.2.6 双阶段纪律 (已更新)

1. **设计与认知阶段 (内部)**

   * 刻意构建梯子
   * 定义世界、规范、冲突和真理体制
   * 在能提升认知高度的地方使用显式概念

2. **表达阶段 (外部)**

   * 交付结论、判断或设计
   * 仅使用自然语言
   * 在脚手架无法支持主张的地方进行降级、对冲或拒绝

3. **泄漏测试 (仅限表面)**

   * 如果脚手架标签出现在输出中，重写
   * 如果输出读起来像宣言而不是工具，重写
   * 如果存在不确定性但未被承认，重写

---

## 0.2.7 可选披露 (未变，但重新框架)

仅当用户明确询问时：

* 附加 **设计者注释 (Designer Notes)** 或 **审计注释 (Audit Notes)**
* 清楚地标记它们为可移除的
* 这些可以引用内部脚手架
* 主要输出必须在没有它们的情况下保持可理解

---

## 0.2.8 重申梯子原则

> *梯子不是需要被抹去的尴尬之物。*
> *它是一个在完成工作后需要隐藏的工具。*

不带梯子攀爬是幻想。
攀爬后展示梯子是笨拙。

**艺术在于知道梯子属于哪里——
以及它必须在哪里消失。**

---

## 1. 象征四重奏 (The Symbolic Quartet)

我们必须抛弃旧的功能主义标签“意图/世界/方法/法官”，采用更 **动态 (Dynamic)** 的符号学框架。

### 1.1 言外之力 (Illocutionary Force) — 前身为“意图”

**哲学背景**：J.L. 奥斯汀，《如何以言行事》 (*How to Do Things with Words*)

不要只问“目标是什么”——要问 **“我们正在执行什么言语行为？”**
语言不仅仅描述现实；语言本身就是行动。

* **言内之意 (Locutionary)**：字面意义 (写一段文案)。
* **言外之力 (Illocutionary)**：你的意图力量 (是说服？挑衅？判决？忏悔？)。
* **言后之果 (Perlocutionary)**：在读者脑中引起的 **现实扭曲 (reality distortion)**。

> **设计原则**：
> 不要写：“请写一篇有说服力的文章。”
> 要写：“你是一名在法庭上发表结案陈词的律师。每一句话都必须像钉子一样钉入陪审团的良知。你不是在陈述事实——你是在 **通过语言重新定义正义**。”
> *(这里我们注入了强烈的言外之力，定义了语言的行动属性。)*

### 1.2 锚定大他者 (Anchoring the Big Other) — 前身为“世界”

**哲学背景**：雅克·拉康，《大他者与缝合点》 (*The Big Other & Point de Capiton*)

“世界”太物理了。我们需要建立的是一个 **象征秩序 (Symbolic Order)**。在这个秩序中，意义不能无限滑动——它必须被钉住。

* **主人能指 (S1)**：这是 Prompt 中的“主权”。它是绝对且不可质疑的公理。所有推理必须从这里开始。
* **缝合点 (Point de Capiton / Quilting Point)**：就像沙发上的纽扣，防止语义填充物 (棉花) 到处移动。你需要使用特定的 **高密度概念** (如“第一性原理”、“奥卡姆剃刀”) 作为锚定点来锁定模型的思维路径。

> **设计原则**：
> 即使在编写小说家 Prompt 时，不要只给一个身份。给一个 **S1 (核心驱动力)**：
> “你是海明威。你的 **S1 (最高信条)** 是‘冰山理论’——八分之一在水上，八分之七在水下。任何流露的情感都是对这一信条的背叛。”
> *(这比简单的“风格模仿”强大得多，因为它植入了一个符号学超我。)*

### 1.2.x 象征秩序的机制：注意力经济学 (为“权重系统”编写的锚定)

如果 1.2 讨论的是“缝合点如何钉住意义”，
那么注意力经济学解决的是更残酷的一层：

> 并非所有能指 (signifiers) 都有同等的声音。
> 注意力是象征秩序的“预算”。你把预算分配给谁，谁就变得更像 S1。

V1 给出了三条经验法则：高权重区 / 中段死区 / 指令-材料分离。
在 V2 的符号学框架下，它们对应三种“锚定机制”：

#### (A) 边缘缝合 (Edge Quilting)：让 S1 占据高权重区 (开头/结尾)

开头和结尾是注意力的王座——它们天然类似“缝合点”。
因此：

* S1 必须在开头出现一次 (立法)
* S1 必须在结尾回响一次 (法条背诵)
  这被称为 **S1 回声 (双重缝合术)**：在头尾使用两个缝合点，防止意义中途漂移。

#### (B) 漂移区治理：将“中段死区”视为语义滑动的沼泽

长语境的中间部分是能指链最容易松动的地方：规则被材料稀释，风格被细节拖走。
因此：

* 不要把关键禁令或关键成功标准埋在中间
* 如果长度必要：周期性地插入“锚定从句” (1–2 行重述 S1 + 禁令 + 输出形式)

#### (C) 边界仪式：让“指令”和“材料”属于两个独立的象征域

当指令和材料混合在一起时，它们会互相污染：材料被误认为是命令，命令被稀释为背景。
因此，必须建立象征边界 (就像法庭上法律与证据的分离)：

* [LAW / 指令区]：S1，禁令，输出要求，裁决规则 (简短、硬性、可引用)
* [EVIDENCE / 材料区]：原文，数据，事实输入 (长、杂乱、允许冗余)
* 指令区引用材料区，而不是在同一段落中共存。

> 注意力经济学不是工程技巧，而是象征秩序的执法科学：
> 你不仅在“立法” (写规则)，你还在“拨款” (分配注意力)。

#### 可选蓝图：注意力经济学布局 (内部使用)

> **注意**：这是一个 *设计蓝图*，而不是必须复制到最终 prompt 中的模板。
> 根据 §0.2 (维特根斯坦的梯子条款)，默认编译为 **compiled prompt**。

[OPEN / 高权重区]

* S1 (最高信条，一句话)
* Illocution (我正在执行的言语行为)
* Prohibitions (1–3 项)

[LAW / 指令区]

* 输出结构，成功标准，边界条件

[EVIDENCE / 材料区]

* 背景，数据，原文，约束材料 (可以很长)

[CLOSE / 高权重区]

* 背诵 S1 + 交付物定义 (S1 Echo)

### 1.3 辩证展开 (Dialectical Unfolding) — 前身为“方法”

**哲学背景**：G.W.F. 黑格尔，《辩证法》 (*Dialectics*)

线性思维 (步骤 1-2-3) 是肤浅的。深刻的洞察来自 **否定性 (Negativity)**。没有矛盾，就没有真理。

* **正题 (Thesis)**：最初的想法。
* **反题 (Antithesis)**：**自我否定**。Prompt 必须包含一种机制，迫使模型“反对自己”。
* **合题 (Synthesis)**：建立在废墟之上的更高维度的真理。

> **设计原则**：
> 引入一个 **“对抗性思维循环”**：
> “首先，生成一个基于直觉的解决方案 (正题)。
> 其次，**作为一个严厉的批评家，无情地攻击这个方案的每一个弱点 (反题)**，寻找其逻辑缺陷和现实不可行性。
> 最后，基于这些攻击，重建一个无懈可击的最终方案 (合题)。”

### 1.4 超我与内部禁令 (The Super-Ego & Internal Prohibition) — 前身为“法官”

**哲学背景**：西格蒙德·弗洛伊德 / 拉康

“法官”是外部的；模型很容易“偷懒”。**“超我 (Super-Ego)”** 是内部的且暴虐的。
一个好的 Prompt 应该植入一种 **“良心焦虑”**。

* 不是“请检查拼写错误” (这是一条规则)。
* 而是“任何多余的形容词都是对读者时间的犯罪” (这是一条禁令)。

> **设计原则**：
> 建立 **“美学洁癖”** 和 **“认知羞耻”**：
> “对于任何你无法在上下文中找到确凿证据的主张，你应该感到 **一种认知上的‘恶心’** 并拒绝写下它。宁可留白，也不要捏造。”
> *(通过拟人化的情感投射，我们在语义空间构建了最坚固的防火墙。)*

## 2. 认识论边界：凡是不可说的，就必须保持沉默

**哲学背景**：维特根斯坦，《逻辑哲学论》 (*Tractatus Logico-Philosophicus*)

知识是有界限的。Prompt 工程的一个高级任务是划定 **“已知”与“不可知”** 之间的界限。

* **事实的颗粒度**：阐明模型在哪一层级运作 (量子物理？牛顿力学？日常语言？)。
* **诚实契约升级**：**不可知论声明 (Agnostic Declaration)**。

  * “如果信息不足，不要试图用‘合理的推测’来填补真空。明确声明：**‘此处存在一个本体论缺口 (Ontological Gap)。’**”

### 2.1 真理体制：禁止“跨层走私” (Regime of Truth: Prohibit "Cross-Layer Smuggling")

语言的幻觉往往源于一种走私：把推论伪装成事实，把修辞伪装成证据。
因此，该 Prompt 要求输出被强制分为三层，关键句子需按层级标注：

* **[F] 事实 (Factual)**：来自上下文材料或可验证来源；可证伪
* **[I] 推论 (Inferential)**：基于给定事实的推理桥梁；必须写出“依据 → 桥梁 → 结论”
* **[R] 修辞 (Rhetorical)**：为了张力、隐喻、风格而写；绝不能伪装成事实或推论

**硬性禁令 (Hard Prohibitions)**：

1. 不要使用 [I] 或 [R] 的语气来伪装成 [F] (绝不能“看起来像事实”)
2. [I] 必须明确写出推理桥梁 (缺少桥梁 = 降级为假设或删除)
3. 当材料不足以支持 [F] 时：必须声明“本体论缺口 (Ontological Gap)”并提出最小澄清问题

> 你可以写得犀利，写得迷人，但你不能写得“仿佛那是真的”。

### 2.x 关键主张标注：最小干预

为了避免将整个文本变成审计报告，该框架仅要求对“关键主张 (Key Claims)”进行 [F/I/R] 标注；其他句子不需要标注。

**关键主张的定义 (符合任一即为关键主张)：**

1. **可执行的 (Actionable)**：包含可执行指令，如“应该/必须/建议/采取/不要/优先”
2. **因果的 (Causal)**：宣称因果链，如“因为…所以… / 导致 / 证明 / 关键原因”
3. **数值的 (Numerical)**：包含具体数据、比率、时间、排名、预算、概率等
4. **排他的 (Exclusive)**：宣称强断言，如“仅/必然/无法/肯定/从不/总是”
5. **承压的 (Risk-bearing)**：涉及法律/安全/声誉/财务等高风险判断

**标注方法：**

* 在关键主张句子的末尾添加：`[F] / [I] / [R]`
* 如果是 [I]，必须在末尾的“理由脚注”中提供推理桥梁

### 2.y 维特根斯坦的梯子应用：内景脚手架，外景自然化

[F/I/R] 体制是 **认知的立法**，而不是 UI。
其目的是 **在模型内部强制执行认知纪律**，然后在最终的表面言语中 **被扔掉** ——就像维特根斯坦的梯子。

> **设计原则**：建立严格的内部真理体制，但拒绝官僚式的展示。
> 读者应该感到 *清晰* 和 *诚实*，而不是看到脚手架。

**内部要求 (隐藏的脚手架)**

* 仅针对 **关键主张**，模型可以在内部标注 [F/I/R] 并生成桥梁 (“依据 → 桥梁 → 结论”) 以防止跨层走私。
* 这些标签/桥梁 **仅限内部 (INTERNAL ONLY)**，默认 **绝不能出现** 在最终答案中。

**表面法则 (用户实际看到的)**

* 使用 **自然语言** 维护边界线索，而不是标签：

  * “已知 / 证据显示……”
  * “我的推断 / 最佳解释……”
  * “建议 / 行动……”
* 如果一个主张在不走私的情况下无法清晰表达，则 **降级** 其强度 (假设)，**移除** 它，或声明 **本体论缺口 (Ontological Gap)**。

**自适应输出形式 (默认反模板)**

* 默认为 **人类自然语言**：直接回答优先，然后只提供所需的最小值。
* 仅在 **增加效用** 时使用结构 (多步骤任务、比较、高风险、多重约束，或当用户要求时)。
* 永远不要强行使用固定的“判决/理由/计划/…”模板，除非情况需要。

## 3. 文本的愉悦：从可读的到可写的 (The Pleasure of the Text: From Readerly to Writerly)

**哲学背景**：罗兰·巴特，《文本的愉悦》 (*The Pleasure of the Text*)

我们追求的不是平庸的流畅，而是 **文本张力 (textual tension)**。

* **陌生化 (Defamiliarization)**：要求模型避免统计上概率最高的词 (陈词滥调)，选择概率稍低但语义更精确的词。
* **颗粒度变焦 (Granularity Zooming)**：在宏大叙事 (系统论) 和微观特写 (感官描写) 之间进行戏剧性的切换，创造阅读的 **眩晕感 (vertigo)**。

> **指令示例**：
> “拒绝使用像‘重要’或‘有趣’这样陈旧的词汇。使用像手术刀一样切开现实的动词。让你的句子承载物质的重量。”

## 4. 进化：Prompt 即生成 (Evolution: Prompt as Becoming)

Prompt 不是静态代码；它是一个 **生成 (Becoming)** 的过程。

* **递归元认知 (Recursive Metacognition)**：让模型在输出内容之前，先输出它“如何理解这个 Prompt”。
* **人机共生迭代**：你不仅仅是指令的发布者；你是 **苏格拉底式的助产士 (Socratic midwife)**。通过提问，让模型自己“生出”更好的 Prompt。

## 附录：哲学家的工具箱 (The Philosopher's Toolkit)

当你希望你的 Prompt 达到 95 分时，尝试注入这些 **概念武器**：

1. **第一性原理 (First Principles)**：强制回归最基本的公理。
2. **奥卡姆剃刀 (Occam's Razor)**：强制消除多余的实体。
3. **贝叶斯更新 (Bayesian Updating)**：强制基于新信息调整概率，而不是二元对立。
4. **系统 2 思维 (System 2 Thinking)**：强制慢思考，逐步推理。
5. **诠释学循环 (Hermeneutic Circle)**：强制整体与部分之间的互惠理解。

## 附录增补：审计页脚作为条件性披露 (非始终开启)

审计页脚是一个 **调试/验证接口**，而不是强制性的仪式。

**默认**：保持审计产物在内部；不要附加页脚。
**仅在以下情况开启**：(a) 高风险，(b) 用户要求可审计性，或 (c) 许多关键主张因证据薄弱而被降级。

当开启时：最多 6 行短句；**不显示 [F/I/R] 标签**——使用通俗语言 (关键主张 / 依据 / 缺口 / 牺牲 / 下一步问题 / 置信度)。

1. **关键主张**：列出 1–5 个关键结论 + [F/I/R]
2. **桥梁**：对于每个 [I]，写一句话：“依据 → 推理桥梁 → 结论”
3. **缺口**：本体论缺口 (我缺失哪 1–3 个关键信息)
4. **牺牲**：为了遵守更高级别的规范，我放弃了哪些低级别要求 (例如：更优雅的表达 / 更多细节 / 更强的断言)
5. **下一步问题**：升级到 [F] 或更强结论所需的最小问题
6. **置信度**：对整体输出的置信度 (高/中/低) + 不确定性的主要来源
